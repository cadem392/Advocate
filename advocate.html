Methodology and Responsible Use

How Advocate handles a denial

Structured as a defensible process: intake → classification → evidence mapping → escalation → drafting → export.

01

Ingest and extract

Parse denial text, insurer references, coverage language, charge details, and dates from uploaded documents.

02

Classify denial

Identify whether the dispute involves medical necessity, coverage terms, documentation failure, coding errors, or utilization review.

03

Detect deadlines

Surface immediate filing windows, missing dependencies, and risk cascades if any stage is missed.

04

Map evidence

Connect clinical notes, prior auth history, plan guidelines, and provider documentation to each viable appeal ground.

05

Generate strategy

Build the attack tree showing which actions to take first, which documents unlock the next stage, and where escalation begins.

06

Draft and export

Produce internal appeals, external review requests, and complaint drafts in a professional, evidence-aware structure.

Common Denial

Medical necessity

Requires progression evidence, specialist notes, and proof conservative treatment failed.

Required Support

Physician addendum

A short clinical statement tying symptoms, failed treatment, and imaging to plan criteria.

Escalation Path

External review

Appropriate when internal appeal fails. Independent reviewers overturn ~40% of denials.

Coding Error

CPT code mismatch

Billing under the wrong CPT code leads to automatic denial. Correcting the code often resolves without appeal.

Insurer Pattern

Template rationale mismatch

Denial language may repeat standardized phrasing even where the case record contains clear supporting evidence.
